Dokumentation - REST-API

Locked routes = Bearer Token REQUIRED!

    GET Routes:

  • GET: /users - för att returnera alla användare.

  • GET: /users/id - för att returnera enskild användare efter ID.

  • GET: /comments - för att returnera alla gästboksinlägg.

  • GET: /comments/id - för att returnera enskild gästboksinlägg.

    POST Routes:

  • POST: /login - för att logga in med hjälp av Insomnia + JSON.

  • POST: /addUser - För att lägga till en användare med hjälp av Insomnia + JSON.

  • POST: /addComment - För att lägga till en kommentar med hjälp av Insomnia + JSON.

    PUT Routes:

  • PUT: /editUser/ID - för att ändra värden på registrerad användare med hjälp av ID + Insomnia + JSON.

  • PUT: /editComment/ID - för att ändra värden på ett gästboksinlägg med hjälp av ID + Insomnia + JSON.

    DELETE Routes:

  • DELETE: /deleteUser/ID - för att radera en specifik användare.

  • DELETE: /deleteComment/ID - för att radera ett specifikt gästboksinlägg.

    JSON-format exempel för Insomnia:

    POST: /login = { "username": "Simon", "password": "Hallstrom" }

    POST: /addUser = { "name": "Simon", "username": "knugen", "password": "knugen", "email": "pwis@knugen.se" }

    POST: /addComment = { "name": "Simon", "message": "Mitt Meddelande" }

    PUT: /editUser = { "name": "Simon", "username": "kungen", "password": "kungen", "email": "pwis@kungen.se" }

    PUT: /editComment = { "name": "Simon", "message": "Mitt Nya Meddelande" }
